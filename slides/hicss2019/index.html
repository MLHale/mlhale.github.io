Our Contribution: SecuWear

Basic idea: Empower software engineers and security analysts with tooling that helps them identify and mitigate security issues in their IoT apps

We've created Audit hooks Applicable to All portions of an IoT/Wearable App

Eventitem:
   created: datetime
   domain: string
   eventtype: string
   event: string
   codereference: string
   data: string
   size: string
   run: integer

Standard Interface: Can be applied to mobile, web, and IoT components in the app. 

Novelty?

Individually audit hooks aren't terribly novel and are already used to wide effect for attack detection (e.g. network monitoring tools).

 

Taken together though, few tools exist to analyze data across various domains of an IoT/Wearable Application.

 

This limits the ability of attack detectors to identify problems that span domains.

Data Collection

Assumptions:

- Implemented the case study app using metawear, android, and a PHP webservice

- Captured data during each of the operational scenarios from the audit hooks and aggregated data into an audit log for the run.

- Ran the scenarios in a real network environment at our University

- Captured data for exactly 2 minutes

- Ran captures 25 times for each scenario

- Each run included the following steps: paired wearable, open android app, loaded advert, displayed temperature information graph

- With the attack scenarios, a malicious advert was delivered

- The malicious adverts exploited a known insecure development practice in android that allows code injection due to not properly sandboxing web-loaded content at run-time.

Data Cleaning and Preparation 

  • Removed malformed network events (only 2 in the  dataset)
  • Filtered out traffic from all devices except the collaborating devices in the app
  • Applied a mutual information feature selection from Peng and Long* to reduce dimensionality and complexity of data
  • Applied 80/20 stratified random sampling from the 75 runs
  • Prepared samples varying the training set size from 5 to 60 within our dataset to examine stability

* H. Peng, F. Long, and C. Ding, “Feature selection based on mutual information: Criteria of Max-Dependency, Max-Relevance, and Min-Redundancy,” IEEE Trans. Pattern Anal. Mach. Intell., vol. 27, no. 8, pp. 1226–1238, 2005.
